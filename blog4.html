The focus of Lecture 10 was responsive web design. Responsive web design responds to the needs of the users and the devices they're using -Ethan Marotte. The layout of a website should be adaptable to a varying set of environments, which are definted by the user accessing the information and the technology displaying the information.

In our video lectures this week, we focused on how to make a website responsive to the technology it is being accessed on and its capabilities. The main distinctions we see in technologies are screen size and input type. A website must adapt its display based on the screen size and orientation. Additionally, a website must consider devices which are controlled by a mouse and keyboard vs. devices controlled by touch screens.

Some of the tips and tricks for responsive web design I found are listed below:

  • Break content up into sections. Then organize these sections based on device [1]
  • Make your call-to-action the center of attention and easy to find on all devices [1]
  • Standardize clickable areas and buttons [1]
  • Use Media Queries to differentiate style based on conditionals [2]
  • Set the viewport [2]

A user accessing this site on a desktop with a mouse as their cursor should have a relatively easy time clicking the intended link. However, a user accessing this site on a mobile device would have a much more difficult time selecting the intended link. As website designers, we must keep in mind that the precision of a mouse versus a typical finger is very different. In order to make the links more accessible on a mobile device, I will add a border and background color and increase the padding and margins.
