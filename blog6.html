During the past week of CS 347, we learned about higher order functions, which are functions that take in other functions as parameters. The main idea with higher order functions is to create abstractions that hide repetitions. One example that we have already used in this class is addEventListener(event, function).

The three higher order functions we learned about in lectures are map, filter, and reduce.

map

The map function is performed on an array and takes another function as a parameter. The function must take a single element as a parameter and return a single element as well. The function is applied to each element in the original array to produce a new array, which is returned.

Example

let nums = [1,2,3,4,5,6];
nums.map(x => x+1);
>> [2,3,4,5,6,7]

Try it for yourself!

filter

The filter function is performed on an array and takes another function as a parameter, however the given function must take a single element as a parameter and return a boolean value. For each element in the original array, if the function returns true, the element is added to a new array, otherwise it is ignored. The new array, containing only elements which result in true when passed into the given function is returned.

Example

let nums = [1,2,3,4,5,6];
nums.filter(x => x % 2 == 0);
>> [2,4,6]

Try it for yourself!

reduce

The reduce function is performed on an array and takes another function and an initial value as parameters. The function must take two elements as parameters and return a single element. For each element in the array, the new initial value is combined with the value returned by the function when given the current initial value and the element. The final initial value is returned.

Example

let nums = [1,2,3,4,5,6];
nums.reduce((sum,x) => sum + x, 0);
>> 21

Try it for yourself!
