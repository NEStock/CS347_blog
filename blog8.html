During the past three weeks of CS 347, we have built two apps using React, the today app, which we finished in Lecture 20, and the notes app, which we started in Leture 21. After completing Lab 20, I am finally feeling more confident in my abilitiy to use React to create web applications.

In the lecture 21 videos, we learned about React Router DOM, a helper library that provides a component Router which enables a React app to have multiple pages loaded directly from a given URL. The implementation is done in the App component via a <Swtich> tag provided by React Router which acts like a switch-statement. The <Route> tag acts like a case in the switch-statement, which re-directs the user based on the URL. The <Redirect> tag acts like a default case.

<Switch>
    <Route exact path="/componentA">
        <ComponentA/>
    </Route>
    <Route exact path="/componentB">
        <ComponentB/>
    </Route>
    <Redirect to="/home"/>
</Switch>

Why would you want to use React Router?

As discussed briefly in the lecture videos, React apps are Single-page applications (SPAs), which "rewrite sections of a page rather than loading entire new pages from a server" [1].

Some of the challenges associated with SPAs discussed in the article I read are maintaining browser history and directly accessing a URL that is not the site's home page. Both of these can be overcome with a router, which allows a user to navigate between pages, builds a browsing history, and keeps the UI state in sync. React Router does exactly this for React apps, solving the problems developers face with SPAs, while still providing all of the advantages of React.
